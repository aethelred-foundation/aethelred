# AIP-0001: AIP Purpose and Guidelines

## Abstract

An Aethelred Improvement Proposal (AIP) is a design document providing information to the Aethelred community, or describing a new feature for the Aethelred protocol. The AIP should provide a concise technical specification and rationale.

## Motivation

We need a standard, transparent process for proposing and tracking protocol changes. This allows the community to participate in protocol governance and ensures all changes are documented and discussed before implementation.

## AIP Types

- **Core**: Protocol-level changes (consensus, PoUW, economics). Require governance vote.
- **Standard**: Interface standards (new message types, SDK APIs, bridge standards).
- **Informational**: Design guidelines, best practices, or general information.
- **Meta**: Changes to the AIP process itself.

## AIP Lifecycle

```
Idea → Draft → Review → Final
```

1. **Idea**: Informal discussion in Discord or GitHub Discussions
2. **Draft**: Author opens a PR with a complete AIP document
3. **Review**: Core team reviews; Core AIPs trigger on-chain governance vote
4. **Final**: Merged after acceptance; implementation begins

## AIP Format Requirements

Every AIP must have:

```yaml
---
aip: <number>          # Assigned by maintainer
title: <Short Title>   # < 80 characters
author: <Name> (@handle)
status: Draft | Review | Final | Withdrawn | Replaced
type: Core | Standard | Informational | Meta
created: YYYY-MM-DD
requires: [AIP-N, ...]  # Optional
---
```

Followed by these sections:
- **Abstract**: 200-word summary
- **Motivation**: Why this change is needed
- **Specification**: Precise technical specification
- **Rationale**: Why this design was chosen over alternatives
- **Backwards Compatibility**: Breaking changes and migration path
- **Security Considerations**: Threat model analysis
- **Implementation**: Link to reference implementation
- **Copyright**: Apache-2.0

## Governance

Core AIPs that affect consensus require an on-chain governance vote via `x/gov` before they become Final. The minimum quorum is 33.4% of staked AETHEL with a 67% Yes threshold.
