Introduction to Aethelred

Aethelred is a decentralized protocol for verifiable AI computation. It enables developers to generate cryptographic proofs that AI inferences were computed correctly, creating trust in AI outputs for regulated industries and high-stakes applications.

Get up and running with your first verified inference in under 5 minutes.

Installation

Install the Aethelred SDK using your preferred package manager:

Python
# Install via pip pip install aethelred # Or with poetry poetry add aethelred
JavaScript
// Install via npm npm install @aethelred/sdk // Or with yarn yarn add @aethelred/sdk

Your First Verified Inference

Here's a complete example of submitting an AI inference request and receiving a Digital Seal:

Python
from aethelred import Client, InferenceRequest # Initialize the client client = Client(api_key="your_api_key") # Create an inference request request = InferenceRequest( model="gpt-4-turbo", messages=[ {"role": "user", "content": "Analyze this medical image..."} ], seal_required=True ) # Submit and wait for verified result result = client.inference.create(request) # Access the AI output print(result.content) # Access the Digital Seal print(result.seal.id) # 0xAE7H...3F2D print(result.seal.verify()) # True
💡 Pro Tip

Set seal_required=True to receive a cryptographic Digital Seal with your inference. Without this flag, you'll receive faster responses but without verification proofs.

API Endpoints

The Aethelred API is organized around REST principles. All requests are authenticated using API keys passed in the Authorization header.

POST /v1/inference

Submit an AI inference request for verified computation. Returns the AI output along with a Digital Seal.

GET /v1/seals/{seal_id}

Retrieve the full metadata and verification status of a Digital Seal by its ID.

POST /v1/seals/verify

Verify the authenticity and integrity of a Digital Seal against the blockchain record.

POST /v1/spatial/attest

Create a Spatial Attestation for physical-world location and environmental data.

Authentication

All API requests require authentication via an API key. Include your key in the request headers:

HTTP Headers
Authorization: Bearer aeth_live_xxxxxxxxxxxxxxxxxxxx Content-Type: application/json
⚠️ Security Warning

Never expose your API keys in client-side code or public repositories. Use environment variables and server-side proxies to keep your keys secure.

Rate Limits

API rate limits depend on your subscription tier:

  • Free Tier: 100 requests/minute, 1,000 requests/day
  • Developer: 1,000 requests/minute, 50,000 requests/day
  • Enterprise: Custom limits based on contract

Rate limit headers are included in all API responses:

Response Headers
X-RateLimit-Limit: 1000 X-RateLimit-Remaining: 847 X-RateLimit-Reset: 1705329127
