Aethelred Glossary

A comprehensive guide to terminology used throughout Aethelred documentation and the verifiable AI ecosystem.

ABCDEFGHIKLMNOPRSTVWZ

A

AETHEL Token TOKEN
The native utility and governance token of the Aethelred network, used for staking, transaction fees, validator rewards, and protocol governance voting.
Attestation
A cryptographic statement signed by a validator confirming that a specific computation was executed correctly according to the Proof-of-Compute protocol.
Air-Gapped Deployment
A deployment configuration where the Aethelred node operates in complete network isolation, suitable for classified or highly sensitive environments.
Audit Trail
An immutable, chronological record of all AI computations, decisions, and verifications stored on the Aethelred network for compliance and forensic analysis.

B

Block Producer
A validator node selected to create the next block in the chain, responsible for aggregating sealed computations and attestations into a verified block.
Byzantine Fault Tolerance (BFT)
The network's ability to reach consensus and continue operating correctly even when some validators are malicious or failing.

C

Computational Provenance
The complete traceable history of a computation, including input data sources, model versions, execution environment, and output attestations.
Consortium
The Aethelred Verification Consortium, a group of organizations collaborating to establish industry standards for verifiable AI computation.
Cross-Domain Solution (CDS)
A secure system that enables verified AI outputs to be shared across different security classification levels while maintaining integrity.
Cryptographic Seal
See Digital Seal. A tamper-evident cryptographic wrapper applied to AI computation outputs.

D

Digital Seal CORE
Aethelred's proprietary cryptographic mechanism that binds AI computation outputs to their provenance, creating an unforgeable certificate of authenticity and integrity.
Delegated Staking
A mechanism allowing AETHEL holders to stake their tokens with a validator node without running their own infrastructure, earning proportional rewards.
Deterministic Execution
The property ensuring that given identical inputs and environment, an AI computation will always produce the same outputs, enabling verification.

E

Enterprise Corridor FEATURE
Dedicated, high-throughput verification channels for enterprise customers with guaranteed SLAs, priority processing, and enhanced privacy features.
Epoch
A fixed time period (currently ~6 hours) during which the validator set remains constant and after which staking rewards are distributed.
Execution Environment Hash
A cryptographic fingerprint of the complete computational environment (hardware, software, libraries) used for an AI computation.

F

Federated Learning
A machine learning approach where models are trained across decentralized data sources without raw data leaving its origin, with Aethelred verifying each training contribution.
Finality
The point at which a sealed computation is irreversibly confirmed on the network and cannot be altered or reverted.

G

Gas
The unit measuring computational effort required to execute operations on the Aethelred network. Gas fees are paid in AETHEL tokens.
Governance Proposal
A formal suggestion for protocol changes submitted by AETHEL holders, requiring community voting to be implemented.

H

Hash Commitment
A cryptographic technique where a hash of data is published first, allowing later verification that the revealed data matches the original commitment.
Hardware Security Module (HSM)
A physical computing device that safeguards cryptographic keys, required for enterprise validator nodes.

I

Inference Verification
The process of cryptographically proving that an AI model produced a specific output given specific inputs, without revealing model weights.
Immutable Ledger
The append-only blockchain storing all sealed computations and attestations, where historical records cannot be modified.

K

Key Ceremony
A formal process for generating and distributing cryptographic keys for network validators, ensuring no single party controls critical secrets.

L

Light Client
A minimal implementation that can verify sealed computations without running a full node, suitable for mobile and embedded applications.
Liveness
The network property ensuring that valid transactions will eventually be processed and sealed computations will eventually be finalized.

M

Merkle Proof
A cryptographic proof allowing efficient verification that a specific computation is included in a block without downloading the entire block.
Model Hash
A unique cryptographic fingerprint of an AI model's weights and architecture, used to verify the exact model version in a sealed computation.

N

Node INFRA
A computer running Aethelred software that participates in the network. Can be a full node (stores all data), validator node (produces blocks), or light node.
Nonce
A unique number used once in cryptographic operations to prevent replay attacks on sealed computations.

O

Oracle
A trusted external data source that provides real-world information to the network, with attestations about data authenticity.

P

Proof-of-Compute CORE
Aethelred's novel consensus mechanism that validates AI computations were executed correctly, combining cryptographic proofs with distributed verification.
Provenance Chain
The linked sequence of seals and attestations documenting every transformation of data from origin through AI processing to final output.

R

Reproducibility Certificate
A sealed attestation that an AI computation can be exactly reproduced given the same inputs and environment specification.
Rollup
A scaling solution that bundles multiple sealed computations into a single on-chain transaction, reducing costs while maintaining security.

S

Slashing
The penalty mechanism where validators lose staked AETHEL for malicious behavior or significant downtime.
Smart Contract DEV
Self-executing code deployed on Aethelred that can trigger automatic verification, manage sealed data, and enforce AI governance rules.
Spatial Attestation
Cryptographic proof of the geographic location where a computation occurred, with ±3 meter accuracy using multi-source verification.
Staking
The process of locking AETHEL tokens to become a validator or support validator operations, earning rewards for network participation.

T

Trusted Execution Environment (TEE)
Secure hardware enclaves (Intel SGX, ARM TrustZone) used by validators to execute sensitive computations with hardware-level isolation.
Timestamp Proof
Cryptographic evidence that a computation occurred at a specific moment in time, resistant to backdating attacks.

V

Validator
A node operator who stakes AETHEL and runs specialized hardware to verify AI computations and participate in consensus.
Verification Proof
The cryptographic evidence that a sealed computation is valid, can be verified by any party without re-executing the computation.

W

Wallet
Software or hardware that manages AETHEL tokens and sealed computation credentials, enabling users to interact with the Aethelred network.
Witness
A validator selected to observe and attest to the correct execution of a computation, providing additional verification guarantees.

Z

Zero-Knowledge Proof (ZKP) CRYPTO
A cryptographic method allowing one party to prove a statement is true without revealing any information beyond the validity of the statement itself.
ZK-SNARK
Zero-Knowledge Succinct Non-Interactive Argument of Knowledge — the specific ZKP construction used in Aethelred's verification proofs.
